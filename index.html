Der Git-Workflow

Als erstes wird ein Repositorium bei GitHub anglegt. Dieses kann dann mit dem Terminalbefehl git clone und der URL des Repositorium auf den eigenen Rechner heruntergeladen werden. Jetzt kann der Code bearbeitet werden. Mit dem Terminalbefehl git add werden die Änderungen in einem staged snapshot vermerkt/vorbereitet und schließlich mit git commit im Lokalen.git zwischengespeichert. Per git push wird dann der Ordner wieder in das Repositorium auf GitHub hochgeladen.
$ git clone https://github.com/aspll/EWP-HA-15.git
$ git add index.html
$ git commit -m "Beschreibung der Änderung"
$ git push origin
