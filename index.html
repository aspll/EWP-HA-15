Der Git-Workflow

Als erstes wird ein Repositorium bei GitHub anglegt. Dieses kann dann mit dem Terminalbefehl git clone und der URL des Repositoriums auf den eigenen Rechner heruntergeladen werden. Jetzt kann der Code bearbeitet werden. Mit dem Terminalbefehl git add werden die veränderten Dateien in einem staged snapshot vermerkt und schließlich mit git commit im lokalen .git-Verzeichnis zwischengespeichert. Per git push werden dann die veränderten Dateien des lokalen Verzeichnis wieder in das Repositorium auf GitHub hochgeladen. Das .git-Repositorium auf dem eigenen Rechner kann per git pull mit dem Repositorium auf GitHub synchronisiert werden und die aktuelle Version bzw. der gewünschte Branch geklont werden.
Git-Befehl Git-Beispiel
$ git clone https://github.com/aspll/EWP-HA-15.git
$ git add index.html
$ git commit -m "Beschreibung der Änderung"
$ git push origin master
$ git pull origin master
